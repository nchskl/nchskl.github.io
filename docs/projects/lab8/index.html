Лабораторная работа №8: Создание клиент-серверного приложения без серверных фреймворков

Цель работы

Цель данной лабораторной работы — создать простое клиент-серверное приложение на Python без использования серверных фреймворков. В ходе работы студент освоит работу с HTTPServer, маршрутизацию запросов, шаблонизатор Jinja2 для отображения данных и архитектуру MVC. Также будет реализована подписка пользователей на валюты и отображение динамики их изменения.

Задачи работы

  1. Создать клиент-серверное приложение с использованием стандартного HTTP-сервера Python.
  2. Реализовать модели предметной области, включая сущности User, Currency, UserCurrency, App, Author.
  3. Применить архитектуру Model-View-Controller (MVC).
  4. Реализовать маршрутизацию HTTP-запросов и отображение данных с помощью шаблонов Jinja2.
  5. Получать данные о курсах валют через функцию get_currencies и отображать их пользователям.
  6. Реализовать функциональность подписки пользователей на валюты и отображение динамики их изменения.
  7. Написать тесты для моделей и серверной логики.

Описание проекта

Проект представляет собой серверное приложение на Python, реализующее функциональность для отображения информации о пользователях и их подписках на валюты. В приложении используется стандартный сервер HTTPServer для маршрутизации запросов и шаблон Jinja2 для рендеринга HTML-страниц.

Основные функции: - Модели: Определяются сущности, такие как User, Currency, UserCurrency, App, Author. - Шаблоны Jinja2: Используются для отображения данных о пользователях и валютах. - Маршруты: Реализованы для отображения информации о пользователях, курсах валют и данных о проекте.

Пример маршрутов

  • /: главная страница с информацией о приложении и авторе.
  • /users: список пользователей.
  • /user?id=...: информация о конкретном пользователе и его подписках.
  • /currencies: список валют с текущими курсами.
  • /author: информация об авторе.
